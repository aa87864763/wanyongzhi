# 具体操作流程

1. 解析命令行参数

2. 初始化计时器

3. 划分4个子范围

4. 启动4个worker协程

5. 协程并行计算素数

6. 素数写入Channel

7. 主协程收集结果

8. 判断所有协程是否完成

9. 所有协程完成后关闭Channel并写入文件

10. 输出统计的信息并结束

# 更新

在原来的基础上采用了生产者-消费者模型了，优化了花费时间。原来是先计算，得到所有素数后一起写入文件中，现在是在计算之前就打开管道等待写入数据。等同时开启协程后实现计算之后立即推送，达到了实时写入文件的功能，耗费时间比原来缩短了2/3。最后打印的内容为：
```bash
所有素数数量为：78498
花费时间 :0.063169秒
```
